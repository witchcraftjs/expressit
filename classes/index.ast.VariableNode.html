Class VariableNode<TValid>

A variable represents just a string value (NOT it's boolean value).

The parser will never return just a variable, they are always wrapped in an

See

  • ConditionNode to give them a boolean value.

A variable might or might not be quoted. If it is, the quote property will contain the quote tokens,

  • ValidToken . While they will always be of the same quote type, one might be valid while the other might not (@see ErrorToken).

value will contain the string value of the variable. The text is not processed and might still contain escaped characters (even if unquoted!) that need to be removed to get the intended value. This is so that the position of the node is accurate.

  • evaluate will take care of unescaping when needed.

If prefixableStrings is true, the prefix property might contain a value token.
